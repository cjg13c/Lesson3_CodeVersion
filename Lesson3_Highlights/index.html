Highlight #1 add

The add command is used to add the file from your workspace to the index. The index is a staging area like a cache that files have to go through before they are finally saved on a repository. they do not yet have all the features that make version control so appealing like all the previous versions being accessable.

Highlight #2 commit

The commit command sends it to the local repository on your machine and saves it as it will be saved with a timestamp and who edited it when it makes its final move to the remote repository accessable by the whole team or just yourself.

Highlight #3 push

The push command is used to push files from the local to the remote repository so that others will be able to do more work on the file and have all the information about who edited the file last and when and all the previous versions as well. The local and the remote repositories are set up exactly the same but the local does not require internet access to require therefore can only be used on your local machine. Once it is pushed to the remote repository over the internet it can be accessed by anyone in the group.
